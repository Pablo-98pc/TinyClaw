Revised Data Flow

  1. Input: User types text or taps push-to-talk mic button
  2. STT (voice only): Whisper base transcribes speech → text
  3. Dispatch: BERT-tiny classifies intent → chat / summarize / task / event
  4. Routing:
    • chat → Chat model responds conversationally
    • summarize → Summarizer processes text
    • task / event → Chat model extracts structured data → saves to SwiftData
  5. Response: Streams back to UI with specialist badge + confirmation for tasks/events
